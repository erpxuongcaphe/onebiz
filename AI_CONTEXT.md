# ONEBIZ ERP - PROJECT HANDOFF CONTEXT

## 1. Project Overview
**OneBiz ERP** is a modern, mobile-first Enterprise Resource Planning interface designed for small businesses in Vietnam. 
**Current Status:** UI Polish Phase Complete. Ready for functional logic expansion.

## 2. Tech Stack & Libraries
- **Core:** React 19, TypeScript, Vite (implied).
- **Styling:** Tailwind CSS (Custom config in `index.html` for `safe-area` and font sizes).
- **Icons:** Lucide React (Stroke width 1.5 or 2 depending on density).
- **Charts:** Recharts (Responsive containers).
- **AI:** Google GenAI SDK (`@google/genai`).
- **Font:** Be Vietnam Pro (Google Fonts).

## 3. Critical Design Rules (DO NOT BREAK)

### A. Mobile First & Navigation
1.  **Bottom Navigation:** On mobile (`lg:hidden`), we use `components/MobileNav.tsx`. Do NOT use a top hamburger menu for main navigation.
2.  **Safe Areas:** The bottom navigation and fixed elements MUST respect iPhone Home Indicator.
    - Use `pb-safe` utility (configured in `index.html`).
    - `viewport-fit=cover` is enabled.
3.  **Sidebar:** Only visible on Desktop (`lg:flex`). On mobile, the "More" button in Bottom Nav triggers the Sidebar as a drawer.

### B. Visual Density (High Density Interface)
This is an ERP, not a landing page. Information density is priority.
- **Font Sizes:** Use granular sizes: `text-[10px]`, `text-xs` (12px), `text-sm` (14px).
- **Padding:** Use tight spacing: `p-1`, `p-1.5`, `p-2`. Avoid `p-4` or `p-6` inside cards unless necessary.
- **Data Display:** Use `tabular-nums` for all numbers/currency.

### C. Color System
- **Light Mode:** Background `bg-slate-50`. Cards `bg-white`.
- **Dark Mode:** Background `bg-slate-950`. Cards `bg-slate-900`.
- **Accents:**
    - Primary: Indigo (`indigo-600`).
    - Success: Emerald (`emerald-600`) - Used for Profits, Stock In.
    - Warning: Amber (`amber-500`) - Used for Low Stock, Pending.
    - Danger: Rose (`rose-600`) - Used for Debts, Out of Stock.

## 4. Component Architecture
- **`App.tsx`:** Layout manager. Handles `activeTab` state and renders `MobileNav` + `Sidebar`.
- **`MobileNav.tsx`:** Fixed bottom bar. Critical for mobile UX.
- **`AIChat.tsx`:** Floating AI assistant.
    - **Positioning:** `bottom-20` on mobile (to clear nav), `bottom-4` on desktop.
- **`Drawer.tsx`:** Used for details (Inventory items, Orders) to prevent page navigation context loss.
- **`Dashboard.tsx`:** Contains KPI Cards and Charts. Note the "Quick Actions" toolbar on mobile.
